## 핵심 디렉토리 상세 설명

### 1️⃣ `backend/stt_core/` - 핵심 STT 모듈

모든 음성 인식 로직을 포함하는 크로스플랫폼 핵심 모듈입니다.

```
stt_core/
├── input_providers/          # 입력 소스 추상화 (Strategy Pattern)
│   ├── base.py              # IAudioInputProvider (인터페이스)
│   ├── bluetooth_provider.py # BluetoothAudioProvider 구현
│   ├── file_provider.py      # FileAudioProvider 구현
│   └── exceptions.py         # ConnectionError, TimeoutError 등
│
├── audio_input/              # 입력 데이터 처리
│   ├── audio_input_module.py # 메인 모듈 (AudioInputModule)
│   ├── audio_data.py         # AudioData 데이터 클래스
│   │   └── class AudioData:
│   │       - audio: np.ndarray
│   │       - sampleRate: int
│   │       - channels: int
│   │       - duration_ms: int
│   │
│   └── validators.py         # 오디오 포맷 검증
│
├── preprocessing/            # 속삭임 특화 전처리
│   ├── preprocessing_module.py   # AudioPreprocessingModule
│   ├── noise_removal.py          # 스펙트럼 감산 (Spectral Subtraction)
│   ├── voice_activity.py         # VAD (낮은 임계값)
│   ├── normalization.py          # 샘플 레이트, RMS 정규화
│   └── filters.py                # 고주파 강화 필터, etc.
│
├── stt_models/               # STT 모델 관리 (Strategy Pattern)
│   ├── base.py              # ISTTModel (인터페이스)
│   ├── whisper_model.py     # WhisperModel 구현
│   ├── vosk_model.py        # VoskModel 구현
│   ├── custom_whisper_model.py  # 파인튜닝 모델용
│   └── model_factory.py     # 팩토리 패턴으로 모델 생성
│
├── result_processing/        # 결과 처리 및 포맷팅
│   ├── result_processing_module.py
│   ├── confidence_calculator.py
│   └── formatters.py
│
└── pipeline/                 # 오케스트레이터
    ├── speech_to_text_pipeline.py  # SpeechToTextPipeline
    └── config.py                   # Config 클래스
```

**특징:**
- ✅ 입력 소스와 무관하게 동작 (파일 or 블루투스)
- ✅ 모듈별 독립적 테스트 가능
- ✅ 각 모듈이 명확한 책임 가짐

## 개발 흐름 예시

### Phase 1: 파일럿 테스트 (1-2주)

```bash
# 1. 프로젝트 클론 및 설정
git clone whispering_voice
cd whisper-voice-conversion

# 2. 개발 환경 설정
bash scripts/setup_dev_env.sh

# 3. 테스트 음성 파일 생성
python scripts/generate_test_audio.py

# 4. 파일럿 테스트 실행
python scripts/run_pilot_test.py

# 5. 상세 테스트
bash scripts/run_tests.sh backend/tests/integration/test_full_pipeline_file.py
```

### Phase 2: 실제 하드웨어 통합 (1주)

```bash
# 1. 블루투스 하드웨어 연결
# (하드웨어팀이 BluetoothAudioProvider 데이터 제공)

# 2. 블루투스 테스트
bash scripts/run_tests.sh backend/tests/integration/test_full_pipeline_bluetooth.py

# 3. 파일 vs 블루투스 일관성 검증
bash scripts/run_tests.sh backend/tests/integration/test_consistency.py
```

### Phase 3: 모바일 앱 개발 (2주)

```bash
# iOS (Xcode 프로젝트)
open mobile/ios/WhisperVoiceApp.xcodeproj

# Android (Android Studio 프로젝트)
# mobile/android 디렉토리를 Android Studio에서 오픈
```

## 주요 설계 원칙

| 원칙 | 구현 방법 |
|------|---------|
| **단일 책임 원칙 (SRP)** | 각 모듈이 명확한 책임 가짐 |
| **개방-폐쇄 원칙 (OCP)** | IAudioInputProvider, ISTTModel 인터페이스로 확장 가능 |
| **의존성 역전 (DIP)** | 추상 클래스/인터페이스에 의존 |
| **파일럿 우선** | FileAudioProvider로 즉시 테스트 가능 |
| **모듈 독립성** | 입력 소스와 무관하게 동작 |
| **설정 분리** | config/ 디렉토리로 환경별 관리 |
